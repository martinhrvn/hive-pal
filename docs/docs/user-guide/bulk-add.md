# Bulk Add

When you do the same thing to many hives — feed the whole apiary, treat for varroa across the board, or introduce a batch of queens — **Bulk Add** lets you stage it once and apply it to every selected hive.

## When to Use It

- Feeding or treating many hives with the same product on the same day.
- Recording the same management action across an apiary.
- Introducing queens to several hives at once.

## Three Types

Bulk Add (`/actions/bulk`) has three tabs:

- **Action** — feeding, treatment, frame changes, or notes.
- **Inspection** — a shared inspection applied to each selected hive.
- **Queen** — add a queen record to each selected hive.

## How It Works

1. Open **Bulk Add** and pick a tab for what you want to add.
2. **Select hives** using the searchable list — choose as many as you like.
3. **Configure** the action, inspection, or queen using the builder for that type, and pick a date.
4. **Add to queue** — the item is staged for the selected hives.
5. Repeat to stage more items (e.g. a feeding *and* a treatment) if needed. You can edit or remove staged items before submitting.
6. **Submit** — everything in the queue is created across the chosen hives. Hive-Pal reports how many succeeded and flags any failures.

## FAQ

**Can I mix actions, inspections, and queens in one submission?**
Yes — stage items from different tabs into the queue, then submit them together.

**What happens if one hive fails?**
Bulk Add reports per-item success and failure counts so you can see exactly which entries didn't go through and retry them.

**Does a bulk inspection score each hive?**
Yes — each hive receives a normal inspection record with scoring, just created in bulk.
